# Artizen Strategy Decision Brief (RC)

## Recommendation (decision-ready)

**Adopt a hybrid staged strategy now:**
1) **Project/artifact-first immediately** (this week) to enter active Artizen flow with the lowest setup friction.  
2) **Fund-first later** (after first cycle evidence + internal alignment), if RC confirms sustained operator capacity and clear domain scope.

This is the best fit with current RC constraints and momentum in the notes: fast execution, lower dependency on external approvals, and direct learning from one full cycle before taking on fund-manager responsibilities.

---

## A) Onboarding strategy options and assessment

## Option 1 — Fund-first

**Pros**
- Positions RC as allocator/curator, not only applicant.
- Can align with RC’s domain strategy (theme/domain-based allocation already discussed in council calls).

**Cons / Risks now**
- Higher setup dependency: requires fund setup process with Artizen team contact and management overhead.
- Requires stronger ongoing governance/ops cadence (review, approvals, comms, season management).
- RC is still closing core accountability work (activities reporting, Bread follow-up, website baseline).

**Dependencies**
- Confirmed operator bandwidth for a full season.
- Domain/fund thesis clarity (what RC funds first).
- External coordination with Artizen fund-setup process.

**Expected lead time (now)**
- **~2–6 weeks** to responsibly launch (given current context and dependencies).

---

## Option 2 — Project/artifact-first

**Pros**
- Fastest path to market presence in current season.
- Lower operational burden than fund management.
- Creates immediate proof set (profile quality, artifact framing, conversion, governance participation).

**Cons / Risks**
- More dependent on campaign execution quality (narrative, artifacts, outreach).
- If RC narrative/evidence is fragmented, conversion may underperform.

**Dependencies**
- Clear project narrative + evidence links.
- Minimum outreach to allied communities.

**Expected lead time (now)**
- **~3–7 days** for profile readiness.
- **~1–2 weeks** for first strong artifact and campaign loop.

---

## Option 3 — Hybrid staged (recommended)

**Why now**
- Matches RC’s actual maturity: active but still consolidating reporting and website credibility layer.
- Preserves optionality: RC can enter now as project, then convert to fund-manager path with evidence.
- Reduces downside: avoids committing to high-ops structure before internal cadence is stable.

**Main risks**
- If phase transition criteria are vague, RC could stay stuck in project-only mode.
- Requires explicit checkpoint for “go/no-go” into fund creation.

**Required checkpoint (after first cycle evidence)**
- RC has: (a) complete profile + artifacts + evidence links, (b) basic website credibility pages live, (c) clear domain thesis, (d) named operators.

**Expected lead time**
- Start in days; decision on fund path after first validated cycle.

## B) Minimum content package for RC Artizen profile/application

## 1) Narrative (minimum viable)
- **What RC is:** cross-network coordination layer linking local nodes/chapters across ecosystems.
- **Why now:** funding fragmentation + need for accountable, domain-relevant deployment.
- **How RC works:** coordinated network support (nodes, funds, domains, activities reporting), not a single isolated project.
- **What this Artizen cycle funds:** one focused deliverable set (avoid broad “everything” framing).

## 2) Evidence links (must include)
- **Activities report evidence** (Karma/activities source pages and summaries).
- **Operational outputs** (meeting decisions, deliverables, dashboards).
- **Public-facing references** (website pages showing current network state).
- **Any prior funding execution evidence** (where available).

## 3) Impact and accountability framing
- Use a simple chain: **Inputs → Activities → Outputs → Near-term outcomes**.
- Explicitly state **what RC can verify now** vs. what is directional.
- Keep claims auditable through links (meeting notes, dashboard data, reports).

## 4) Specific role of Bread Coop + activities report + 2026 vision
- **Bread Coop role:** explain as current strategic funding/governance entanglement channel, with governance/accountability implications.
- **Activities report role:** establish RC’s execution credibility (not only intent).
- **2026 vision role:** show trajectory and scope boundaries (what RC will and will not do this cycle).

---

## C) Website scoping implications for Artizen credibility

RC website must quickly expose **verifiable, structured proof**. Priority is not design complexity; it is trust clarity.

## Must expose first
1. **Nodes/chapters map/list** (who is active, where, status).  
2. **Funds/programs overview** (what mechanisms are active/planned).  
3. **Domains focus** (what RC prioritizes now, and why).  
4. **Activities/reporting** (recent actions and accountability trail).  
5. **Bread coordination section** (funding/accountability context).  
6. **Integrations matrix** (Karma, Green Goods, Coop, Bloom, etc. with status).

## Data fields to render first (from current RC OS data model)
- **Nodes:** id, name, geography/bioregion, network affiliation, status, evidence/report link.
- **Funds:** id, mechanism type, status (active/pipeline), chain/tool, partner networks.
- **Domains:** domain name, rationale, active initiatives, associated funds.
- **Activities:** date, activity type, owner, linked evidence, related node/fund/domain.
