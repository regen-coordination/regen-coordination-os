# AI Features

Coop uses AI to help people sort, shape, and review context inside the browser. The product promise
is not "replace the group." It is "help the group find signal faster without giving up control."

## What AI Does In Coop

Today the AI layer is aimed at practical support work:

- identify promising opportunities in captures
- score how well something fits the coop's purpose
- cluster related findings
- draft brief summaries and next-step suggestions
- prepare review digests and action proposals for humans

## Why The Browser Matters

Coop tries to do useful work locally first instead of assuming a cloud model is the product. The
browser can run AI analysis on the device, with graceful fallback when local hardware cannot support
the heaviest models.

That matters because it keeps the privacy story coherent:

- raw context does not need to leave the browser just to be analyzed
- the default path keeps working closer to the user's device
- the group can review AI output before it becomes shared state

## Human Review Is Part Of The Feature

Coop's AI is most useful when paired with human review. The system is built around three ideas:

- advisory outputs that stay informational
- proposals that humans can approve
- bounded auto-run only where the policy model explicitly allows it

In other words, the AI layer helps people work through messy context. It is not meant to blur the
difference between a suggestion and a group decision.

## What The Product Does Not Promise

- perfect classification on every capture
- autonomous publishing of shared knowledge by default
- a cloud-first assistant that sees everything automatically
- a magical replacement for editing, judgment, or trust inside a group

The value is in legible assistance, not in pretending the hard parts of coordination disappeared.
